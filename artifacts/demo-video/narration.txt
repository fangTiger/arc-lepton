大家好，我直接演示一下我们做的 SIGNAL LEDGER。

它解决的问题很具体：AI Agent 做交易研究时，经常要买新闻、链上数据、社交情绪这些付费数据。那它该买哪个、花多少钱、每次调用有没有账，这就是我们想补上的闭环。

这里我已经用 OKX Wallet 登录好了。钱包不是装饰，它会绑定我的研究记录、预算和支付流水。

现在我进 Research 页面，输入一个问题，比如 Should I buy PEPE this week，然后给它一个小预算。

注意，这个预算真的会限制 Agent。它不能无限调接口，而是要自己判断：先看便宜的情绪数据，还是需要新闻、巨鲸转账、K 线信号。

点开始之后，左边是 Agent 的实时日志。你可以看到它一边思考，一边调用工具，不是一次性吐一段答案。

中间这个 TX Feed 是重点。每一次数据源调用，都会生成一笔 payment record。右边是预算面板，可以看到已经花了多少 test USDC，还剩多少，调用了几次。

这也正好贴合 Lepton 的主题：不是人手动点付款，而是 Agent 在预算里自己决定哪些资源值得买。

我们这里用 Arc 来记录支付回执。confirmed 的记录可以进 explorer 查，所以它不是假装有支付，而是把研究、数据购买、预算消耗和链上凭证连成一条线。

研究完成后，打开完整报告页。这里有最终结论、调用过的数据源、每次花费和 tx hash。

下面还可以继续追问，比如问这个结论最大的风险是什么。它会基于同一份报告继续分析，而不是重新开一个孤立聊天。

所以 SIGNAL LEDGER 本质上是一个给 AI Agent 用的可审计研究终端：用户给预算，Agent 自主购买数据，支付用 USDC 计量，过程在 Arc 上留下记录，最后产出一份可以继续追问的报告。
